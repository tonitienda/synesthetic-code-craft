# 02 — Treatment

## Working title

Backpropagation: How Neural Networks Learn From Mistakes

## Act I title

The First Artificial Neuron

## Audience

Software engineers and technically curious viewers.

They may know programming, but we do not assume machine learning knowledge.

## Core promise

Backpropagation will eventually feel less mysterious if we first understand what a network is made of:

small units connected by weights.

## Act I conceptual goal

The viewer should understand:

- a perceptron is a simple artificial neuron
- it receives input numbers
- each input has a weight
- weights control importance
- the neuron combines weighted inputs
- the neuron produces a decision
- changing weights changes the decision
- this is the seed of learning
- one perceptron is limited because it can only draw a linear boundary

## Act I emotional goal

The perceptron should feel elegant and powerful, but also incomplete.

The act should create curiosity, not closure.

## Style

Calm.
Precise.
Minimal.
Beautiful.

The visual style should be closer to a quiet engineering explanation than an AI hype video.

## Story summary

We open with the word backpropagation, but immediately rewind.

Before deep networks, there was a simpler dream:

> Could a machine learn from examples?

A single artificial neuron appears.

It receives inputs.

Weights appear on the input connections.

Signals pass through the weighted connections.

The neuron combines them and emits a decision.

Then one weight changes.

The decision changes.

This is the first intuition of learning.

Finally, the same neuron becomes a straight decision boundary on a two-dimensional graph.

The line moves as the weights change.

This is powerful, but limited.

Some problems cannot be solved with one line.

## Ending hook

A single perceptron can learn.

But one perceptron can only draw one kind of boundary.

So the next question is:

> what happens if we connect many perceptrons together?
