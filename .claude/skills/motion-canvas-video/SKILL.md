# Motion Canvas Video Skill

Use this repo-local workflow when creating or extending a Motion Canvas video in this repository.

## Style notes

- Use high-contrast backgrounds and a small palette per video.
- Prefer visual metaphors and progressive disclosure over dense text.
- Keep scene timing aligned with the script timestamps.
- Make videos as one continuous flow instead of multiple disjointed scenes. Avoid abrupt cuts and transitions.

## Narration

- Narration must ONLY talk about the subject being explained (e.g. Docker, git), never about the animation itself. Never narrate what the visuals are doing ("let's put the terminal inside the machine", "watch this box move") — if a visual relationship matters, express it with motion (wrap, contain, connect) and let the animation speak for itself. Litmus test: the narration should still make complete sense as a podcast, with the video turned off.
