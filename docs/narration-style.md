# Narration Style Guide

Narration should feel calm, clear, and precise.

The target viewer is technically curious, but may not know machine learning yet.

## Principles

- Use short sentences.
- Introduce one new idea at a time.
- Prefer concrete language before abstract language.
- Avoid hype.
- Avoid pretending something is simple when it is not.
- Let important ideas breathe.
- End beats with a question when possible.

## Avoid

- "simply"
- "obviously"
- "just" when it hides complexity
- exaggerated claims
- fast influencer-style pacing
- too many terms in one sentence

## Preferred rhythm

1. Show the object.
2. Name it.
3. Explain what it does.
4. Show what changes.
5. Create the next question.

Example:

```text
This is an artificial neuron.
It receives input values.
Each input is a number.
But not every input matters equally.
That is why the connections have weights.
```

## For TTS

- Use punctuation to create natural pauses.
- Avoid very long sentences.
- Prefer spelling out historical dates in narration, for example: "the late nineteen fifties".
- Keep mathematical notation mostly on screen, not in narration.
